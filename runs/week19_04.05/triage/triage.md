## 8. Schema Improvement Suggestions

1. **`prior_verdict` and `prior_score`** — for `already_analyzed=true` records, surface these in the B2 schema directly so the refresh-vs-monitor decision can be made without re-opening prior FINAL-REPORT.json files. Would also enable the agent to flag verdict-flip candidates (CAT this cycle) more reliably.
2. **`price_change_since_last_analysis`** — explicit %-move from prior analysis date would make the >20% refresh trigger mechanical rather than judgmental. Currently relies on b1_reason narrative or web search. Add a small price-fetch step after B1 for already-analyzed names.
3. **`peer_overlap`** — short field listing 1-2 already-analyzed peers covering similar exposure. Would make the "DSV.CO already covers freight forwarding" or "ADP already covers payroll" reasoning explicit and auditable rather than buried in `reason_for_action`.
4. **`days_since_last_analysis`** — derived integer instead of leaving the agent to compute from `analysis_date`. Minor but tightens reproducibility and lets the >180-day staleness rule fire automatically.
